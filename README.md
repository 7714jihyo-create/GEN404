# 🏥 종합병원 404 (미완성)

---

## 🎮 게임 개요

**종합병원 404**는 2~5명이 의사가 되어 이상한 환자들을 진단하는 웹 파티 게임입니다.  
AI가 플레이어의 진단서를 바탕으로 웹소설/메디컬드라마 형식의 결과 보고서를 생성하며,  
Firebase Realtime DB를 통해 여러 기기에서 실시간으로 플레이 가능합니다.

---

## ✅ 구현된 기능

### 실시간 멀티플레이
| 기능 | 내용 |
|------|------|
| 방 만들기 | 4자리 랜덤 코드 자동 생성, 방장 권한 부여 |
| 방 참가 | 코드 입력 후 즉시 참가 |
| 대기실 | 참가자 실시간 표시, 코드 복사 공유 |
| 실시간 동기화 | Firebase Realtime DB / BroadcastChannel 폴백 |
| 유저 세션 분리 | LocalStorage 기반 고유 플레이어 ID |
| 진단서 분리 | 각자 자기 것만 작성, 다른 플레이어 제출 현황 확인 |
| 동시 결과 출력 | 방장이 계산 후 DB 저장 → 전원 화면에 동시 렌더링 |

### UI 디자인
- **흰 배경 + 수술복(민트/청록) 포인트** 컬러 테마
- 타이틀/라운드/시뮬레이션 화면은 다크 딥그린 배경 유지 (병원 밤 분위기)
- 카드형 레이아웃, 결과 유형별 상단 컬러 강조선

### 게임 시스템 (총 11개 화면)
| 화면 | 내용 |
|------|------|
| 타이틀 | 게임 소개 |
| 로비 | 방 만들기 / 참가 |
| 대기실 | 참가자 현황, 방장 시작 버튼 |
| 라운드 시작 | 환자 등장 연출 |
| 초진 차트 | 환자 정보 공개 |
| 진단서 작성 | 60초 타이머 + 내 카드만 수정 가능 |
| 제출 대기 | 제출 현황 실시간 표시 |
| AI 시뮬레이션 | 로딩 연출 |
| 결과 보고서 | 웹소설형 AI 보고서 + 점수 |
| 최종 점수 | 순위표 + 등급 부여 |
| 패배자 이벤트 | 최하위자 특별 초진 차트 생성 |

### AI 보고서 엔진 (완전 재설계)
- **처방 내용을 서사 트리거로 사용** — 유저가 쓴 치료법이 스토리 첫 문장에 직접 인용
- 딱딱한 섹션 헤더 없음 — 자연스럽게 이어지는 150자 이상 메디컬 드라마 형식
- 6가지 결과별 완전히 다른 2개 이상의 스토리 템플릿
- 빈 입력 시 "방치된 환자" 황당 해프닝 자동 생성

### 환자 데이터
- 6개 직관적 종족: 사물 / 동물 / 사람 / 로봇 / 외계인 / 귀신
- 행동 위주 질환: "와이파이 공유기인데 비밀번호 물어보면 욕을 함" 등 시각적 증상

## 📡 멀티플레이 작동 방식

```
[실제 Firebase 연결 시]
Firebase Realtime DB → 실시간 onValue 리스너 → 모든 기기 동시 업데이트

[Firebase 연결 불가 시 (자동 폴백)]
BroadcastChannel API → 같은 기기 여러 탭 간 실시간 동기화
(같은 기기에서 탭을 여러 개 열어 멀티플레이 테스트 가능)
```

### Firebase 실제 연동 방법
`js/firebase.js` 상단의 `firebaseConfig`를 본인의 Firebase 프로젝트 설정으로 교체:
```js
const firebaseConfig = {
  apiKey: "실제-API-KEY",
  databaseURL: "https://your-project-rtdb.firebaseio.com",
  // ...
};
```
Firebase 콘솔 → Realtime Database → 보안 규칙을 개발 중에는 아래로 설정:
```json
{ "rules": { ".read": true, ".write": true } }
```
